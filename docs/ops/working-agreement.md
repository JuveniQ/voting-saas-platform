# Working Agreement

## Communication

- use `WhatsApp` for fast coordination and blockers
- use `GitHub` for all lasting decisions, scoped work, review comments, and release notes
- summarize any WhatsApp decision into GitHub within 24 hours
- never share secrets, tokens, or client-sensitive data in WhatsApp
- never paste sensitive project context into consumer AI tools

## Delivery

- standup update is due by `09:30 SAST` on weekdays
- first code review should happen within one business day
- if a task is blocked for more than one business day, mark it `Blocked` and explain why
- keep one major in-progress task per person
- do not merge red builds

## Quality

- every change goes through a pull request
- every pull request must link to an issue
- UI changes need screenshots or video
- behavior changes need docs updates where relevant
- schema or API changes must update the contract docs
- AI-assisted work must follow the rules in `docs/ai`
- AI output never replaces human review or repo validation

## Collaboration

- ask for help early instead of carrying silent risk
- prefer small PRs over long-running branches
- disagree in docs and comments, not in silence
- the person who opens a review request stays responsible for getting it merged safely
