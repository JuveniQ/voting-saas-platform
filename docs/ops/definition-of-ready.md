# Definition of Ready

An issue is `Ready` only when all of the following are true:

- the problem statement is clear
- there is exactly one owner
- acceptance criteria are written
- dependencies or blockers are listed
- the correct area and priority are set
- a Penpot link is attached if the change affects UI
- API or schema impact is noted if relevant
- the work is small enough to fit in one pull request or has already been split
