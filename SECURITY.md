# Security Policy

## Supported Scope

This repository contains the public codebase and documentation for `voting-saas-platform`.

Do not report security issues in:

- public GitHub issues
- public pull requests
- GitHub Discussions
- WhatsApp groups

## Reporting a Security Issue

Until a dedicated security inbox is created, report security concerns privately to the CTO and CEO through the company's private channel.

Your report should include:

- summary of the issue
- affected route, area, or component
- possible impact
- reproduction steps if safe to share
- logs, screenshots, or traces if available

## Internal Handling

- log the issue privately first
- assess severity and blast radius
- avoid public discussion until mitigation is in place
- create follow-up fixes, tests, and runbook updates after triage

For internal response steps, see:

- [`docs/security/threat-model.md`](./docs/security/threat-model.md)
- [`docs/security/fraud-playbook.md`](./docs/security/fraud-playbook.md)
- [`docs/ops/incident-response-runbook.md`](./docs/ops/incident-response-runbook.md)
